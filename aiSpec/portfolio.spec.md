# Page Spec — Portfolio (portfolio.html)

## Page goal
Showcase two selected projects with a short explanation and a link to their GitHub repositories.

## Target audience
Recruiters, coaches, and technical reviewers.

## Sections

### 1) Header
- Reuse the global header (logo, name, navigation links)

### 2) Page title
- Title: "Portfolio"
- Short introductory sentence explaining that these are selected projects

### 3) Projects section
Display exactly two project cards.

Each project card must contain:
- Project title
- One image (screenshot or placeholder)
- Short text description (2–3 lines max):
  - What the project is about
  - What technologies were used
  - What was learned
- A link to the GitHub repository
  - Opens in a new tab

### 4) Footer
- Reuse the global footer
- Social links: LinkedIn, GitHub, Email

## Content constraints
- Keep descriptions concise and professional
- No long paragraphs
- Focus on clarity and readability

## Styling requirements
- Card-based layout
- Responsive grid (1 column on mobile, 2 on desktop)
- Images should have rounded corners
- Consistent spacing between cards
